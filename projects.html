My Projects

Projects i do just for... fun.

Solpkg

I think this was one of the first projects I was actually proud of.
I got the idea from an already existing wrapper for pacman, but since I was using
Solus, there was nothing that already existed for EOPKG. So I made exactly that,
solely for the "I need this". I can also say that half of my Python knowledge
came from reading the sys.argv documentation and trying to apply that on the code.
Even tought i'm on Arch now, I'm still mantaining the project, and I'm planning
some changes to the functioning of the code (I'm just lazy to do that, lol.).

Github: Jocadbz/solpkg

Pacy

The second project I developed. After getting used to Solpkg, I came to Arch
And found that the ability to just type "solpkg i package" was gone forever.
So I spended time to develop a "copy" of Solpkg, and Pacy was born. The idea
of the name came from pac (pacman) + y (py), it's pretty smart, isn't it?
Pacy is still actively developed, and the version 4 was just released. Also,
this project was one of my first sucessful experiences with Arch packaging.

Github: Jocadbz/pacy

Classh

Github: Jocadbz/classh

Power mode

With the freedesktop foundation trying to improve the Linux desktop by creating
tools that cover use cases, we got Power Profiles, the best library to manage CPU
profiles. GNOME and Plasma, he two giants of the Linux Desktop adopted them, and
Settings are available on both control centers. If your desktop don't yet have GUI
support, you can use the command-line tool, "powerprofilesctl". And yes, it does
it's job well, but still, some missing features and overral improvements are missing,
but that isn't their focus, since they want every DE adopt it. But since I prefer
using the CLI tool, i've developed powermode, which is basically a wrapper for the
CLI tool. It has "Visual" anwsers to commands, and improved usability. This tool
is mantained, but it reached a state where i don't see any visble improvements I can implement.

Github: Jocadbz/power-mode
